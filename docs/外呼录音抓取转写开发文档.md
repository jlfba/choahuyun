# 外呼录音抓取转写开发文档

## 目标

实现一个可以登录外呼系统、逐页抓取带录音的通话记录、下载录音、将录音转文字并合并导出到同一张表的自动化工具。

## 输入与输出

### 输入

- 站点地址：`https://chaohuzhineng.com`
- 登录方式：人工首次登录，脚本保存会话
- 目标字段：
  - 用户信息
  - 公司信息
  - 主叫
  - 被叫
  - 客户姓名
  - 客户公司
  - 标记
  - 通话时间
  - 接通时间
  - 挂断时间
  - 状态
  - 拨号时间
  - 录音
- 筛选条件：只抓取有录音的数据

### 输出

- 原始录音文件
- 每条录音的转写文本
- 合并后的 `xlsx/csv` 总表

## 方案说明

### 技术选型

- Python 3.12
- Playwright：页面自动化
- Requests：携带会话下载录音
- Pandas/OpenPyXL：生成表格
- OpenAI 音频转写接口：将录音转成文字

### 关键设计

#### 1. 登录

不直接在脚本里固化账号密码，而是首次打开浏览器由人工完成登录，再保存 `storage_state`。这样更安全，也更适合有验证码或二次验证的系统。

#### 2. 列表抓取

优先按照截图对应的 Element UI 风格默认选择器实现，同时把选择器下沉到配置文件，便于在页面微调时快速修正。

#### 3. 录音下载

按以下顺序尝试：

1. 直接读取录音单元格里的链接地址
2. 监听浏览器下载事件
3. 点击后从页面弹出的 `audio/source` 节点中提取地址
4. 对 `blob:` 音频通过浏览器上下文读取字节并落盘

#### 4. 文件命名

录音文件基础命名规则为 `主叫_被叫`。若发生重名，自动追加 `__2`、`__3` 等后缀，避免覆盖。

#### 5. 转写回填

转写完成后，根据抓取时保存的主叫/被叫匹配键，将文本回写到对应记录，并导出最终总表。

## 风险与应对

- 页面实际 DOM 与截图不一致
  - 通过 `selectors` 配置项调整
- 录音不是直接下载，而是弹窗播放器
  - 已加入弹窗音频和 `blob:` 抽取方案
- 同号码组合重复出现
  - 自动补充文件名序号，表里保留匹配键和录音文件名
- 转写接口额度或网络问题
  - 单条失败不中断全量流程，失败状态会记录到导出表

## 批量日期范围抓取 (v0.2.0)

### 需求

支持按日期范围批量抓取多天数据，而非只能逐日手动运行。

### 方案

新增 atch_scrape.py，接收开始日期与结束日期参数，依次对范围内每一天执行完整的单日流水线：

1. 调用 uto_scrape.py 抓取当日通话记录与录音
2. 调用 	ranscribe_sensevoice.py 对已下载录音进行语音转写
3. 调用 daily_task.clean_data 清洗转写文本、过滤无效通话
4. 调用 clean_audio.py 清理孤立与重复音频文件

### 用法

```powershell
python batch_scrape.py 2026-07-10 2026-07-22
python batch_scrape.py 2026-07-10 2026-07-22 --force
python batch_scrape.py 2026-07-10 2026-07-22 --skip-transcribe
```

### 关键设计

- 每日独立处理，已有转写结果默认跳过，避免重复劳动
- --force 可强制重新抓取已存在数据的日期
- --skip-transcribe 仅抓取与下载，不进行转写（节省时间）
- 结束时输出每日处理状态汇总
